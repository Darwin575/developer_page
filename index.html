Abstract

This comprehensive study addresses operational inefficiencies and paper-intensive workflows in BJMP Region 10's document routing system through the development of a sophisticated web-based Document Tracking System (DTS). The research demonstrates how modern digital solutions can transform government document management by providing flexible submission methods, secure tracking capabilities, and comprehensive oversight tools. Developed using Agile Scrum methodology, the system features multiple document submission options, QR code integration for alternative receipt methods, robust security protocols, and advanced reporting capabilities that collectively enhance document workflow efficiency while addressing resource consumption concerns.

  • Comprehensive Solution: Multi-faceted system addressing document submission, tracking, security, and reporting needs
  • Flexible Implementation: Provides digital alternatives while accommodating existing workflows and user preferences
  • Enhanced Security: Multi-layered security approach with OTP verification, RBAC, and encrypted token management

Introduction

The research addresses critical challenges identified through extensive interviews with BJMP jail officers, revealing significant operational bottlenecks in document routing processes. These challenges include excessive paper consumption from high-volume daily document routing, inadequate tracking mechanisms leading to document loss, inefficient storage and retrieval systems, and limited oversight capabilities for monitoring document status and workflow progress. The study recognizes that while paper reduction is a primary concern, the solution must also address broader workflow inefficiencies, security vulnerabilities, and the need for comprehensive document lifecycle management.

  • Regulatory Alignment: Supports Republic Act 11032 and DICT's digital transformation initiatives for government modernization
  • Operational Impact: Addresses workflow inefficiencies, improves document traceability, and reduces resource consumption
  • Strategic Implementation: Designed for RICTMD integration with scalable architecture for broader organizational adoption
  • Evidence-Based Design: Solution developed from direct stakeholder feedback and comprehensive needs assessment

Methodology

Development Approach

Agile Scrum Framework with continuous BJMP stakeholder feedback integration is employed throughout the development lifecycle. Key phases include Sprint Planning, Development, Review, and Deployment.

Technology Stack

  • Frontend: HTML5, CSS3, Bootstrap, jQuery
  • Backend: PHP 7.4+, MySQL 8.0
  • Security: RBAC, OTP, CSRF Protection
  • Development Tools: XAMPP, VS Code, MySQL Workbench

Data Collection Methods

  • Surveys: BJMP staff workflow analysis
  • Interviews: Stakeholder requirements gathering
  • SUS Testing: Usability evaluation metrics
  • System Logs: Performance and usage analytics

System Architecture

Three-Tier Architecture

  • Presentation Layer: Responsive web interface with dynamic content rendering and intuitive user experience design.
  • Application Layer: Business logic processing, authentication management, QR code generation, and intelligent routing algorithms.
  • Data Layer: MySQL database managing document metadata, user profiles, audit trails, and system configurations.

Key Components

  • Authentication Module: Multi-factor authentication with OTP
  • Document Router: Intelligent routing based on document types
  • QR Generator: Unique identifiers for document tracking
  • Notification System: Real-time status updates
